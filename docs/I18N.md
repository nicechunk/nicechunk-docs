# Nicechunk I18N Specification

## Goals

Nicechunk uses page-scoped, single-language loading. The client stores the user's selected language and the matching language JSON version locally per page. On startup, each page loads only its own locale file, for example `/civilization/locales/en.json` or `/contracts/locales/en.json`.

`/mainnet.json` is a high-frequency runtime configuration file for game, world, chain, and resource data only. It must not contain locale indexes, dictionaries, language metadata, or page text.

## Default Language And Text Rules

English is the default language for all project files and UI fallbacks.

Rules for future work:

- Comments must be written in English.
- HTML fallback text must be written in English.
- JavaScript fallback strings must be written in English.
- Placeholder text, aria labels, titles, tooltips, HUD labels, and button labels must default to English.
- Non-English user-facing text must be placed only in the locale dictionary source and generated page locale JSON files.
- Chinese text must not appear in source code, docs, HTML, CSS, configuration, or runtime manifests outside the locale dictionary source and generated locale JSON files.
- New pages must expose text through dictionary keys from the first implementation.

## Supported Languages

Default language and display order:

1. `en` - English
2. `es` - Spanish
3. `fr` - French
4. `de` - German
5. `ja` - Japanese
6. `ru` - Russian
7. `ko` - Korean
8. `zh-Hant` - Traditional Chinese
9. `zh-Hans` - Simplified Chinese

## Files

- `src/i18n.js`: runtime loader, translator, DOM binding, and local cache logic.
- `scripts/update-locales.mjs`: source of truth for dictionary content and page-locale generation.
- `public/<page>/locales/{language}.json`: one generated JSON file per language for each page or app surface.
- `src/i18n.js`: shared loader used by pages that declare `data-i18n-scope`.
- Page-specific loaders such as `docs/docs.js`, `contracts/contracts.js`, and `world_rule/i18n.js`: custom loaders for pages with specialized rendering needs.
- `public/mainnet.json`: high-frequency game runtime index. It is not an i18n manifest.

## Mainnet Index

The static development version lives at `public/mainnet.json` and is served as `/mainnet.json`.

Current top-level sections:

- `schemaVersion`: index schema version.
- `app`: app id, app version, minimum client version, target network.
- `updatePolicy`: how clients should compare versions and cache files.
- `world`: base world settings that can later move to chain-managed config.
- `chain`: current Solana cluster, program ids, and chain-side table references.
- `resources`: reserved area for future game asset and manifest versions.

Forbidden in `public/mainnet.json`:

- `i18n`, `*I18n`, `locale*`, `languageOrder`, `dictionary`, or localized page text.
- Per-page language file URLs, content hashes, byte sizes, or cache keys.
- Website-only content that the game runtime does not need.

`scripts/update-locales.mjs` writes `public/mainnet.json`, but the generated mainnet object must be built from game/runtime sections only.

## Local Storage

The shared page runtime uses page-scoped keys:

- `nicechunk.language`: selected language code.
- `nicechunk.<page>.locale.version.<lang>`: cached build version for that page and language JSON.
- `nicechunk.<page>.locale.data.<lang>`: cached JSON string for that page and language.

Startup flow:

1. Read `nicechunk.language`, defaulting to `en`.
2. Detect the page i18n scope, usually from `<html data-i18n-scope="...">`.
3. Resolve that scope to its locale base, for example `/home/locales`, `/docs/locales`, or `/civilization/locales`.
4. If the page-scoped cached build version matches the current app build, parse the page-scoped cached JSON and use it.
5. If local cache is missing or stale, fetch only `/<page>/locales/<language>.json`.
6. Store the fetched page dictionary and build version locally.

## Development Rules

- Do not hardcode user-visible text.
- Add static text with `data-i18n="namespace.key"`.
- Add placeholders with `data-i18n-placeholder="namespace.key"`.
- Add aria labels with `data-i18n-aria-label="namespace.key"`.
- Add runtime text with `t("namespace.key", params)`.
- Call `await initI18n()` in every new page entry before rendering translated runtime content.
- Add every new key to every language in `scripts/update-locales.mjs`.
- Run `node scripts/update-locales.mjs` after dictionary edits.
- Do not import or fetch all language files at startup.

## Example

```html
<button data-i18n="login.startGame">Start Game</button>
<input data-i18n-placeholder="login.playerNamePlaceholder" />
```

```js
import { initI18n, t } from "./src/i18n.js";

await initI18n();
statusLine.textContent = t("login.status.connecting", { wallet: "Phantom" });
```
